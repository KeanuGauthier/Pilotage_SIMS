4.3

Les scénarios définis précédemment ont été traduits en scripts Python à l’aide de Selenium WebDriver, qui permet de piloter un navigateur comme le ferait un utilisateur, localement ou à distance (Selenium, 2024).

Les résultats des tests ont ensuite été transmis vers Zephyr, un plugin intégré à Jira permettant de construire des plans de test détaillés et de suivre leurs exécutions. L’utilisation des API Zephyr permet d’automatiser une partie de ces échanges avec l’outil de gestion des tests (SmartBear, s. d.-b).

Les scripts ont ensuite été préparés pour une intégration dans Jenkins. Jenkins Pipeline permet de modéliser des chaînes d’exécution automatisées et reproductibles, notamment pour les étapes de test et de déploiement (Jenkins, s. d.).


4.4

Une première version a été développée en Python, à partir du module CSV, qui permet de lire et d’écrire des données tabulaires dans des formats compatibles avec des outils comme Excel (Python Software Foundation, s. d.).


5.1

Les scénarios retenus ont été formalisés dans Zephyr sous la forme de plans de test détaillés. Ce choix est cohérent avec l’objectif de structuration documentaire des tests logiciel, qui vise à rendre les contrôles plus explicites, reproductibles et exploitables (ISO/IEC/IEEE, 2022).


5.2

Les plans de test ont été traduits en scripts Python utilisant Selenium WebDriver, afin d’exécuter automatiquement les actions prévues dans les interfaces web de Jira et Confluence (Selenium, 2024).


6.3

Les résultats obtenus peuvent être confrontés à l’analyse de l’existant présentée précédemment, ainsi qu’aux principes généraux du test logiciel. Les tests de non-régression visent précisément à vérifier qu’une modification n’a pas introduit de défaut dans des fonctionnalités déjà existantes (ISTQB, s. d.).

La mise en place des plans de test dans Zephyr répond directement à cette limite. Elle rapproche le fonctionnement de l’équipe d’une démarche plus structurée de gestion des tests, conforme aux principes de formalisation et de suivi décrits dans les référentiels de test logiciel (ISO/IEC/IEEE, 2022).
